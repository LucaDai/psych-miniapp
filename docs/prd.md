# 心理自测小程序 MVP — 产品需求文档（PRD）

> 版本：v1.0  
> 状态：已确认  
> 更新日期：2026-06-06

---

## 1. 产品概述

### 1.1 产品定位

| 项目 | 说明 |
|------|------|
| 产品名称 | 心理自测小程序（暂定） |
| 产品类型 | 轻量心理/性格自测工具 |
| 目标用户 | 对自我认知、情绪状态、性格倾向有兴趣的普通微信用户 |
| 核心价值 | 快速完成测试 → 获得可读的结果解读 → 查看不可删除的历史记录 |
| 第一版规模 | 上线 3 个测试 |

### 1.2 明确非目标

- 不做 AI 分析
- 不做付费功能
- 不做社交功能（分享、评论、排行榜等）
- 不做心理疾病诊断
- 不考虑高并发优化
- 不获取用户昵称、头像，仅使用微信 `openid` 标识用户

### 1.3 合规要求

全站统一使用以下表述：

- 「自测」「仅供参考」「不构成医疗建议」
- 不提供疾病名称、治疗方案、用药建议
- 结果页必须展示固定免责声明

## 2. 用户角色

| 角色 | 说明 | 使用端 |
|------|------|--------|
| C 端用户 | 微信用户，浏览测试、答题、查看结果与历史 | 微信小程序（原生） |
| 管理员 | 内容配置人员，维护测试、题目、选项、结果规则 | React Admin |

MVP 仅支持**单一管理员账号 + 密码登录**，不做角色权限细分。

## 4. 管理端功能需求

### 4.1 登录

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| AD-01 | 账号密码登录 | 单一管理员账号 | P0 |
| AD-02 | Token 鉴权 | 登录后访问管理接口需携带 Token | P0 |

### 4.2 测试管理

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| QZ-01 | 测试列表 | 展示全部测试（含草稿、已上架、已归档） | P0 |
| QZ-02 | 创建测试 | 标题、简介、预计时长、排序、封面图 URL | P0 |
| QZ-03 | 编辑测试 | 修改基本信息 | P0 |
| QZ-04 | 软删除 | 删除后 C 端不可见，数据保留，历史记录仍可关联 | P0 |
| QZ-05 | 上架 | 校验题目、选项、结果规则完整性后上架 | P0 |
| QZ-06 | 下架 | 将已上架测试改为草稿，C 端不可见 | P0 |
| QZ-07 | 封面图 | 数据库预留 `cover_image_url`；MVP 可填 URL 或留空 | P0 |

### 4.3 测试详情页（题目与规则管理）

管理端以**测试详情页**为内容配置入口，在同一页面内通过 Tab 或分区管理：

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| QM-01 | 基本信息区 | 编辑测试标题、简介、封面、时长、排序 | P0 |
| QM-02 | 题目管理区 | 题目 CRUD：题干、排序 | P0 |
| QM-03 | 选项管理区 | 在题目下管理选项：文本、分值、排序 | P0 |
| QM-04 | 结果规则区 | 规则 CRUD：分数区间、结果标题、结果描述 | P0 |
| QM-05 | 上架操作 | 详情页内提供上架 / 下架按钮 | P0 |

### 4.4 题目与选项

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| QS-01 | 题目 CRUD | 每题包含题干、排序序号 | P0 |
| QS-02 | 选项 CRUD | 每题至少 2 个选项，含文本与分值 | P0 |
| QS-03 | 题型 | MVP 固定为单选题 `single_choice` | P0 |

### 4.5 结果规则

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| RR-01 | 计分方式 | 总分区间计分：`total_score = Σ(所选选项分值)` | P0 |
| RR-02 | 规则配置 | 每条规则包含：最低分（含）、最高分（含）、结果标题、结果描述 | P0 |
| RR-03 | 区间唯一 | 同一测试内分数区间不可重叠 | P0 |
| RR-04 | 区间覆盖 | 上架前须校验区间覆盖所有可能总分 | P0 |
| RR-05 | 快照写入 | 用户提交时，将命中规则的标题与描述写入答题记录快照字段 | P0 |

## 5. 计分与结果规则（业务逻辑）

### 5.1 计分公式

```
total_score = sum(selected_option.score for each question)
```

### 5.2 结果匹配

在 `result_rule` 中查找满足以下条件的唯一规则：

```
min_score <= total_score <= max_score
```

若无法命中唯一规则，提交失败并返回错误。

### 5.3 结果快照（方案 B）

提交成功时，在 `test_attempt` 中写入：

- `result_title`：命中规则当时的标题
- `result_description`：命中规则当时的描述
- `result_rule_id`：命中规则 ID（仅供参考，展示以快照为准）

## 6. 用户流程

### 6.1 C 端主流程

```
打开小程序
  → 微信静默登录（openid）
  → 首页（测试列表）
  → 测试详情页
  → 开始测试
  → 答题页（一题一屏，中途退出丢弃）
  → 提交答案
  → 结果页（含免责声明）
  → 历史记录列表 / 历史详情
```

### 6.2 管理端主流程

```
登录管理后台
  → 测试列表
  → 测试详情页
      ├── 编辑基本信息
      ├── 管理题目
      ├── 管理选项（含分值）
      └── 配置结果规则
  → 校验完整性
  → 上架测试
```

## 7. MVP 范围

### 7.1 纳入第一版

| 端 | 范围 |
|----|------|
| 小程序 | 登录、首页、测试详情、一题一屏答题、提交、结果页、历史列表与详情 |
| 管理端 | 登录、测试 CRUD、软删除、上下架、题目/选项/规则管理（测试详情页内） |
| 后端 | 上述全部 API、总分计分、结果快照、记录持久化 |
| 数据 | 8 张表，详见 `schema.md` |
| 内容 | 预置 3 个测试 |

### 7.2 明确排除

| 功能 | 说明 |
|------|------|
| 多维度计分 | 如 MBTI 四维度，二期考虑 |
| 多选题 / 跳题逻辑 | 二期考虑 |
| 答题草稿 / 断点续答 | 中途退出直接丢弃 |
| 历史记录删除 | 不提供 |
| 用户昵称头像 | 仅 openid |
| AI 分析、付费、社交 | 见非目标 |
| 图片上传服务 | MVP 封面仅支持填写 URL |
| 多管理员 / RBAC | 单账号 |
| 数据统计看板 | 二期考虑 |

## 8. 非功能需求

| 类别 | 要求 |
|------|------|
| 部署环境 | 本地开发（Spring Boot + MySQL + 微信开发者工具） |
| 接口风格 | REST API，JSON 格式 |
| 性能 | 常规请求响应，无特殊优化 |
| 安全 | 管理端 Token 鉴权；C 端接口校验用户身份；用户只能访问自己的历史记录 |
| 数据完整性 | 软删除测试不破坏历史记录；结果快照保证历史可读性 |

---

## 9. 上架校验规则

测试上架前，后端须校验：

1. 测试未软删除
2. 至少有 1 道题目
3. 每道题目至少有 2 个选项
4. 结果规则区间不重叠
5. 结果规则覆盖该测试所有可能总分（根据选项分值组合计算 min/max）
